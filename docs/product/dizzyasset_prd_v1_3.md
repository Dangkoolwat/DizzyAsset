# DizzyAsset PRD v1.3

## 1. 제품 개요

**DizzyAsset**은 Final Cut Pro 사용자를 중심으로 한 맥용 편집 리소스 관리 앱이다. 사용자는 내장 디스크, 외장 SSD, 다운로드 폴더 등 여러 위치에 흩어진 효과음, 음악, 밈 영상, 짤, 영상 소스 등을 앱에 추가하고, 디지에셋은 이를 복사 없이 인덱싱하여 검색, 미리보기, 태깅, 프로젝트별 정리를 제공한다.

핵심 포지션은 다음과 같다.

> **Final Cut Pro를 위한 위치 독립 에셋 허브**

또는 더 직관적으로:

> **내 맥과 외장 SSD에 흩어진 에셋을 하나로 묶고, 파컷에서 바로 쓰게 해주는 시스템**

---

## 2. 문제 정의

영상 편집자와 음악 작업자는 효과음, 밈 영상, 짤, 음악 소스 등 수많은 리소스를 반복적으로 사용한다. 하지만 실제 작업 환경에서는 다음과 같은 문제가 발생한다.

### 2.1 에셋이 여러 위치에 흩어져 있음

- 내장 디스크, 외장 SSD, 다운로드 폴더, 프로젝트 폴더 등에 리소스가 분산되어 있다.
- 파일을 매번 Finder에서 찾아야 한다.
- 같은 파일이 여러 위치에 중복 저장되는 경우가 많다.

### 2.2 Final Cut Pro의 사운드 효과 검색/관리 한계

- Final Cut Pro는 자체 음향효과를 제공하지만 검색과 분류가 편집자 관점에서 충분히 편하지 않다.
- 외부 효과음이나 밈 소스는 사용자가 직접 정리해야 한다.
- Final Cut Pro의 사운드 라이브러리와 외장 리소스 폴더를 연결하는 방법을 모르는 사용자가 많다.

### 2.3 인터넷 배포 리소스의 비일관성

- 개인이 만든 효과음/밈 영상 파일은 파일명과 폴더 구조가 일관적이지 않다.
- 같은 의미의 파일도 `funny`, `comedy`, `웃김`, `fail`, `실패`처럼 제각각 저장되어 있다.
- 사용자가 모든 파일을 기억하거나 직접 태그해야 하는 것은 큰 부담이다.

### 2.4 바로 쓰기 어려운 파일 품질

- 효과음 앞뒤에 무음 구간이 포함되어 있는 경우가 많다.
- 타임라인에 올렸을 때 타이밍이 밀린다.
- 사용자가 직접 편집해서 잘라내야 한다.

---

## 3. 제품 목표

DizzyAsset의 목표는 편집자가 에셋을 기억하고 찾아다니는 시간을 줄이고, 원하는 리소스를 빠르게 찾아 Final Cut Pro에서 즉시 사용할 수 있게 만드는 것이다.

### 핵심 목표

1. 파일 위치와 상관없이 에셋을 하나의 라이브러리처럼 관리한다.
2. 파일명, 폴더명, 메타데이터, 분석 결과를 기반으로 자동 태그와 추천 카테고리를 제공한다.
3. 사용자가 직접 만든 카테고리/프로젝트에 드래그하여 논리적으로 정리할 수 있게 한다.
4. Space 키 기반의 빠른 미리보기와 검색 경험을 제공한다.
5. Final Cut Pro에서 바로 사용할 수 있도록 드래그 앤 드롭 및 사운드 라이브러리 연결을 지원한다.
6. 무음 구간 감지 및 클린 버전 생성을 통해 효과음을 바로 쓰기 좋게 다듬는다.

---

## 4. 핵심 사용자

### 4.1 주요 사용자

- Final Cut Pro 기반 영상 편집자
- 유튜브/쇼츠/릴스 제작자
- 게임 영상 편집자
- 밈 영상 편집자
- 사운드 효과를 자주 사용하는 음악/영상 작업자

### 4.2 사용자 특징

- 수많은 효과음, 짤, 밈 영상, 음악 소스를 보유하고 있다.
- 외장 SSD에 리소스를 저장하는 경우가 많다.
- Finder 폴더 정리에 한계를 느낀다.
- 빠른 미리보기와 검색을 중요하게 여긴다.
- Final Cut Pro에서 바로 쓸 수 있는 흐름을 원한다.

## 5. 제품 컨셉

DizzyAsset은 파일을 물리적으로 정리하는 앱이 아니라, 파일 시스템 위에 편집자용 의미 레이어를 얹는 앱이다.

### 핵심 개념

> **파일은 어디 있어도 되고, 정리는 디지에셋에서 한다.**

### 개념적 구조

- 실제 파일: 내장 디스크, 외장 SSD, 다운로드 폴더 등 여러 위치에 존재
- DizzyAsset DB: 파일 위치, 메타데이터, 태그, 카테고리, 프로젝트, 분석 정보를 저장
- 앱 UI: 사용자에게 하나의 통합 라이브러리처럼 표시

### Smart Asset Link

DizzyAsset의 내부 에셋은 단순 파일 경로가 아니라 다음 정보를 가진 스마트 링크이다.

- 원본 파일 위치
- 파일 해시
- 파일 타입/길이/크기
- 태그
- 카테고리
- 프로젝트 연결
- AI/분석 결과
- 사용자 수정 기록

## 6. 정보 구조

### 6.1 물리적 파일과 논리적 분류의 분리

DizzyAsset은 물리적 폴더 구조와 논리적 태그/카테고리/프로젝트 구조를 분리한다.

예:

    실제 위치:
    /Volumes/ExternalSSD/SFX/bonk.mp3

    DizzyAsset 논리 분류:
    - 코믹 효과음
    - 실패
    - 타격음
    - 쇼츠용
    - 프로젝트 A

파일은 이동하지 않지만, 앱 안에서는 여러 카테고리와 프로젝트에 동시에 속할 수 있다.

### 6.2 데이터 저장 위치

DizzyAsset은 원본 파일, 앱 내부 데이터, 작업 공간을 분리하여 관리한다.

#### 원본 리소스

원본 파일은 사용자가 보관한 위치에 그대로 둔다.

    /Volumes/ExternalSSD/SFX/boom.wav
    ~/Downloads/meme_sound.mp3

DizzyAsset은 원본 파일을 이동하거나 수정하지 않고, 위치와 메타데이터만 인덱싱한다.

#### 앱 내부 데이터

앱의 데이터베이스, 설정, 북마크, 인덱스 정보는 macOS 표준 앱 데이터 위치에 저장한다.

    ~/Library/Application Support/DizzyAsset/

포함 항목:

    Database/
    Settings/
    Index/
    Bookmarks/
    AIAnalysis/

역할:

- 앱의 기억
- 데이터베이스
- 설정
- 보안 접근 북마크
- 인덱싱 상태
- 내부 메타데이터

#### DizzyAsset 작업 공간

DizzyAsset이 생성하는 작업본과 파생 리소스는 기본적으로 사용자 홈의 DizzyAsset 폴더에 저장한다.

    ~/DizzyAsset/

포함 항목:

    Derived/
      Trimmed/
      Converted/
      Proxy/

    Generated/
      Preview/
      Export/

    Analysis/
      Waveforms/
      Speech/
      Vision/

    Temp/

역할:

- 무음 제거본
- 변환본
- 프록시
- 생성된 미리보기
- 파형 분석 결과
- 분석 산출물
- 내보내기 결과물
- 임시 작업 파일

#### 작업 공간 이동

`~/DizzyAsset/`은 기본 작업 공간일 뿐이며, 설정에서 다른 위치로 변경할 수 있다.

예:

    /Volumes/EditSSD/DizzyAsset/
    /Volumes/NAS/DizzyAsset/

작업 공간 변경 시 사용자는 다음 중 선택할 수 있어야 한다.

    [기존 작업 공간 파일 이동]
    [새 위치만 사용]
    [취소]

#### 원칙

- 원본 파일은 사용자의 세계에 그대로 둔다.
- Application Support는 앱의 기억을 저장한다.
- `~/DizzyAsset/`은 DizzyAsset의 기본 작업 공간이다.
- 작업 공간은 사용자 지정 가능하다.
- 원본 파일과 작업본/파생 리소스는 명확히 분리한다.

## 7. 주요 기능

## 7.1 파일/폴더 추가 및 인덱싱

### 설명

사용자가 파일 또는 폴더를 드래그 앤 드롭하면 DizzyAsset은 내부의 미디어 파일을 스캔하고 에셋 후보로 등록한다.

### 지원 대상

- 오디오: mp3, wav, aiff, m4a 등
- 영상: mp4, mov 등
- 이미지/짤: png, jpg, gif 등

### 동작

1. 사용자가 파일/폴더를 드래그 앤 드롭한다.
2. 앱은 미디어 파일을 스캔한다.
3. 중복을 체크한다.
4. 메타데이터를 추출한다.
5. 파일명/폴더명 기반 태그 후보를 생성한다.
6. 목록에 등록한다.

### 결과 표시 예시

    248개 파일 발견
    213개 등록 완료
    35개 중복 제외

---

## 7.2 중복 탐지

### 설명

중복 탐지는 두 가지 방식으로 제공한다.

1. 파일/폴더 등록 시 자동 중복 탐지
2. 설정 또는 관리 도구에서 실행하는 전체 중복 재검사

DizzyAsset은 중복을 탐지하고 설명하지만, 원본 파일 삭제는 자동으로 하지 않는다.

---

### 7.2.1 등록 시 자동 중복 탐지

사용자가 파일 또는 폴더를 추가할 때 DizzyAsset은 즉시 중복 여부를 검사한다.

#### 탐지 기준

1차 체크:

- 동일 경로
- 파일 크기
- 확장자
- 수정일

2차 체크:

- 부분 해시
- 전체 해시, 필요 시

부분 해시는 대용량 파일을 고려하여 파일 앞/중간/끝 일부를 사용한다.

예:

    파일 앞 8KB
    파일 중간 8KB
    파일 끝 8KB

전체 해시는 중복 후보로 판단된 경우 백그라운드에서 계산한다.

#### 사용자 안내 예시

    248개 파일 발견
    180개 신규 등록
    42개 이미 등록된 경로
    20개 동일 파일 중복
    6개 실패

#### 기본 동작

- 동일 경로는 재등록하지 않는다.
- 동일 내용 파일은 기본적으로 제외한다.
- 사용자는 필요 시 기존 에셋 열기 또는 별도 등록을 선택할 수 있다.

---

### 7.2.2 전체 중복 재검사

사용자는 설정 또는 관리 화면에서 전체 라이브러리 중복 검사를 실행할 수 있다.

#### 목적

- 이전에 등록된 에셋 중복 정리
- 외장 SSD/다운로드 폴더 정리
- 경로는 다르지만 내용이 같은 파일 확인
- 오래된 인덱스 재검증

#### 실행 위치

    Settings
      → Library / Storage
      → Scan Duplicates

또는:

    Tools
      → Find Duplicates

#### 탐지 범위

- 전체 라이브러리
- 특정 폴더/저장소
- 특정 카테고리/프로젝트

#### 결과 표시

    전체 에셋: 12,430개
    중복 후보: 342개
    동일 내용 확정: 218개
    확인 필요: 124개

#### 사용자 액션

- 기존 에셋에 위치 추가
- 별도 에셋 유지
- 무시
- Finder에서 보기
- 중복 목록 내보내기

---

### 7.2.3 중복 유형

#### 경로 중복

같은 경로의 파일이 이미 등록된 경우.

    /Volumes/SFX/boom.wav
    /Volumes/SFX/boom.wav

처리:

- 재등록하지 않음
- 기존 에셋 표시
- 변경 사항만 재스캔 가능

#### 내용 중복

경로는 다르지만 파일 내용이 같은 경우.

    ~/Downloads/boom.wav
    /Volumes/SFX/boom_copy.wav

처리:

- 동일 내용 파일로 표시
- 기본적으로 신규 등록 제외
- 사용자가 별도 등록 가능

---

### 7.2.4 설계 원칙

- 등록 시 중복 탐지는 빠르고 자동이어야 한다.
- 전체 중복 재검사는 사용자가 명시적으로 실행한다.
- 대량 전체 검사는 백그라운드에서 실행한다.
- 원본 파일은 자동 삭제하지 않는다.
- DizzyAsset은 중복을 알려주되, 물리 삭제 결정은 사용자가 한다.

## 7.3 AI 추천 카테고리 및 태그

### 설명

DizzyAsset의 핵심은 편집자 언어 기반 자동 분류이다. AI는 최종 판단자가 아니라 초안을 만드는 역할이다.

기존의 단순 태그 예시는 실제 편집자의 검색 방식과 거리가 있다. DizzyAsset의 추천 태그는 일반 객체 분류 수준이 아니라, 영상 편집자와 Final Cut Pro 사용자의 검색 패턴, 효과음 라이브러리 명명 관습, 숏폼/밈/예능 편집 문화를 반영해야 한다.

### 원칙

> **AI는 초안을 만들고, 최종 구조는 사용자가 만든다.**

> **태그는 AI 연구용 분류가 아니라 편집자가 실제 검색하는 언어여야 한다.**

### 편집자 언어 기반 태그

추천 태그는 다음 기준을 우선한다.

- 상황
- 감정
- 연출
- 용도
- 플랫폼
- 리듬
- 장르
- 밈 문맥
- 방송/예능 표현
- 한국어/영어 혼합 검색어

### 한글/영어 병기 구조

DizzyAsset의 기본 추천 태그와 카테고리는 한국어와 영어를 함께 저장하고 표시할 수 있어야 한다.

예:

    예능/Funny
    긴장/Tension
    타격/Impact
    전환/Transition
    공포/Horror
    쇼츠/Shorts
    브이로그/Vlog
    리액션/Reaction

DB에서는 `name_ko`, `name_en`, `normalized_name`, `aliases`를 분리해 저장하는 것을 권장한다.

### 기본 추천 태그 데이터베이스

초기 추천 태그 데이터베이스는 단순히 모델 출력에 의존하지 않고, 인터넷 기반 편집 문화와 효과음 라이브러리 관습을 참고해 심도 있게 구축한다.

참고 대상:

- YouTube 편집 문화
- Shorts/TikTok/Reels 편집 패턴
- 밈 영상 편집 패턴
- 방송/예능 효과음 관용어
- Motion Graphics 용어
- Final Cut Pro / Premiere 사용자 커뮤니티 표현
- 효과음 라이브러리 naming convention
- 한국어/영어 혼합 편집 용어

### 태그 계층 예시

#### 감정 계열

    긴장/Tension
    웅장/Epic
    감동/Emotional
    공포/Horror
    슬픔/Sad
    따뜻함/Warm

### AI의 역할

AI는 임의로 태그를 남발하지 않고, 분석 결과를 편집 언어 사전에 매핑한다.

입력 예:

    file: fail_bonk_funny.mp3
    folder: meme/fail
    sound: short impact

출력 예:

    코믹/Funny
    실패/Fail
    타격/Impact
    짧음/Short Hit
    쇼츠/Shorts

### 추천 태그 상태

AI 추천 태그는 기본적으로 확정 태그가 아니라 추천 상태로 표시한다.

상태:

- suggested
- confirmed
- rejected
- user

사용자가 승인하거나 직접 수정한 태그는 사용자 확정 태그로 저장한다.

### 설계 원칙

- AI 추천은 최대한 기존 태그/카테고리에 매핑한다.
- 새 태그는 자동 확정하지 않는다.
- 신뢰도가 낮은 추천은 숨기거나 낮은 우선순위로 표시한다.
- 사용자 수정 이력은 이후 추천 품질 개선에 활용한다.

## 7.4 사용자 카테고리 및 드래그 태깅

### 설명

사용자는 왼쪽 패널에서 자신만의 카테고리나 프로젝트를 만들 수 있다. 중앙 목록에서 일부 에셋을 선택한 후 왼쪽 카테고리로 드래그하면 해당 에셋에 카테고리 태그가 자동으로 부여된다.

### 카테고리 깊이

카테고리는 기본 2단계, 최대 3단계 깊이를 가진다.

DizzyAsset은 무제한 폴더 트리를 지양하며, 편집자가 빠르게 탐색할 수 있는 얕은 의미 구조를 우선한다.

권장 깊이:

    기본: 2단계
    최대: 3단계

예:

    효과음/SFX
      └─ 예능/Entertainment
          └─ 타격/Impact

또는:

    음악/Music
      └─ 긴장/Tension
          └─ 상승/Riser

### 단계별 의미

- 1단계: 큰 용도
- 2단계: 편집 상황/감정/장르
- 3단계: 세부 표현
- 4단계 이상 금지

### 예시 구조

    효과음/SFX
      ├─ 전환/Transition
      ├─ 타격/Impact
      ├─ 리액션/Reaction
      └─ 예능/Entertainment

    음악/Music
      ├─ 긴장/Tension
      ├─ 감동/Emotional
      ├─ 웅장/Epic
      └─ 브이로그/Vlog

    밈/Meme
      ├─ 실패/Fail
      ├─ 놀람/Surprise
      ├─ 황당/Awkward
      └─ 리액션/Reaction

### 편집 가능 항목

사용자는 사용자 카테고리에 대해 다음 작업을 수행할 수 있어야 한다.

- 카테고리 추가
- 카테고리 이름 변경
- 카테고리 이동
- 카테고리 병합
- 카테고리 삭제
- 카테고리 순서 변경

### 드래그 태깅 동작

1. 사용자가 카테고리 생성
2. 중앙 리스트에서 에셋 선택
3. 왼쪽 카테고리로 드래그
4. DB에 관계 저장
5. 파일은 이동하지 않음

### 중요한 원칙

- 파일 이동 없음
- 파일 복사 없음
- DB 관계만 추가
- 하나의 에셋은 여러 카테고리에 속할 수 있음
- 카테고리 삭제는 원본 파일 삭제가 아님

삭제 안내 문구:

    카테고리를 삭제해도 원본 파일은 삭제되지 않습니다.
    에셋과 카테고리 연결만 제거됩니다.

### AI 추천 카테고리 정책

AI가 추천하는 카테고리도 최대 3단계 깊이 제한을 따라야 한다.

- AI 추천은 최대 3단계까지만 생성한다.
- 이미 존재하는 카테고리에 우선 매핑한다.
- 새 카테고리는 추천 상태로만 표시하고 자동 확정하지 않는다.

## 7.5 중앙 검색/필터

### 설명

중앙 영역은 단순 리스트가 아니라 에셋 탐색 엔진이다.

### 검색 대상

- 파일명
- 태그
- 카테고리
- 프로젝트
- 분석 텍스트
- 전사 텍스트

### 필터

- 태그별 검색
- 최신 등록순
- 이름순
- 길이순
- 파일 타입별
- 외톨이 보기

### 외톨이 정의

외톨이는 아직 정리되지 않은 에셋을 의미한다.

조건:

- 사용자 카테고리 없음
- 프로젝트 연결 없음
- 사용자 확정 태그 없음

### UX 예시

    [검색창] [태그 필터] [정렬: 최신 등록순]

    bonk.mp3     [코믹][실패][짧음]     0.5s
    wow.mov      [밈][리액션]          2.1s

## 7.6 빠른 미리보기

### 설명

편집자는 많은 에셋을 빠르게 훑어야 하므로 미리보기 속도가 핵심이다.

### 주요 동작

- 방향키: 목록 이동
- 방향키 이동 중 연속 미리보기 가능
- 오른쪽 상세 패널에서 미리보기 버튼 제공

### UX 목표

- 딜레이 최소화
- Finder Quick Look처럼 자연스러운 사용감
- 대량 에셋 탐색에 적합한 키보드 중심 흐름

## 7.7 바로 가기 돋보기 / Quick Peek

### 설명

Quick Peek은 DizzyAsset의 핵심 생산성 UX이다. 사용자는 편집 중 단축키로 작은 오버레이 검색창을 띄우고, 에셋을 검색/미리보기/선택할 수 있다.

### 사용자 흐름

1. Final Cut Pro 작업 중 단축키 입력
2. DizzyAsset Quick Peek 오버레이 표시
3. 검색어 입력
4. 방향키로 결과 이동
5. Space로 미리듣기
6. Enter 또는 드래그로 사용

### 키보드 동작

- ↑ / ↓: 결과 이동
- Space: 미리보기
- Enter: 선택
- Esc: 닫기

### 목표

> **편집 도중 1초 안에 원하는 에셋을 꺼내는 도구**

## 7.8 Final Cut Pro 연동

### 설명

DizzyAsset은 Final Cut Pro용 위치 독립 에셋 허브로 동작해야 한다. 따라서 찾은 에셋은 Final Cut Pro에서 바로 사용할 수 있어야 한다.

### 주요 기능

#### 1. Final Cut Pro로 드래그 앤 드롭

DizzyAsset 목록 또는 Quick Peek에서 에셋을 드래그하여 Final Cut Pro 타임라인에 놓으면 실제 파일 경로 기반으로 삽입된다.

#### 2. Final Cut Pro 사운드 라이브러리 연결 제안

앱은 Final Cut Pro의 사운드 라이브러리 위치와 DizzyAsset에서 관리하는 리소스 위치를 비교한다.

둘이 다르면 다음과 같이 제안한다.

    Final Cut Pro 사운드 효과 폴더와
    디지에셋 관리 폴더가 다릅니다.

    디지에셋 에셋을 Final Cut Pro에서도 사용할 수 있도록
    소프트 링크로 연결할까요?

    [연결하기] [나중에]

### 원칙

- 원본 파일을 이동하지 않는다.
- 원본 파일을 복사하지 않는다.
- Final Cut Pro가 볼 수 있도록 연결만 만든다.
- 필요 시 Final Cut Pro 재시작 안내를 제공한다.

## 12. 비기능 요구사항

### 성능

- 대량 파일 등록 시 UI가 멈추지 않아야 한다.
- 인덱싱은 백그라운드에서 처리한다.
- 검색은 즉시 반응해야 한다.
- Space 미리보기는 지연이 거의 없어야 한다.

### 안정성

- 원본 파일은 기본적으로 수정하지 않는다.
- 외장 드라이브 연결 해제 시 링크 깨짐 상태를 표시한다.
- DB 손상을 대비해 백업을 유지한다.

### 프라이버시

- 기본 분석은 로컬에서 수행한다.
- 외부 LLM 사용 시 명시적 사용자 동의를 받는다.
- 분석 결과는 로컬 DB에 저장한다.

### 확장성

- 분석기는 교체 가능한 구조로 설계한다.
- Apple 분석, 로컬 LLM, 클라우드 LLM을 분리된 Provider로 구현한다.

## 13. 성공 기준

### 사용자 경험 기준

- 사용자는 파일 위치를 기억하지 않아도 된다.
- 원하는 효과음을 빠르게 검색하고 미리들을 수 있다.
- 찾은 파일을 Final Cut Pro에 바로 사용할 수 있다.
- 흩어진 외장/내장 리소스가 하나의 라이브러리처럼 느껴진다.

### 정량 지표 예시

- 파일 등록 성공률
- 중복 탐지 정확도
- 평균 검색 응답 시간
- 미리보기 시작 지연 시간
- 등록 후 태그가 하나 이상 붙은 에셋 비율
- 사용자가 직접 분류한 에셋 비율
- Final Cut Pro 드래그 사용 횟수

## 16. 설정(Preferences)

설정은 최소한으로 시작하되, 사용자가 불안해할 지점과 워크플로우에 직접 영향을 주는 항목만 제공한다.

### 16.1 라이브러리 / 데이터

- 인덱싱 데이터 위치 표시
  - `~/Library/Application Support/DizzyAsset/`
- DB 백업 생성/복원
- 캐시 정리 (썸네일/파형)
- 손상된 링크 재검사

### 16.2 에셋 추가

- 기본 동작
  - (●) 원본 위치 유지 (소프트 링크)
  - ( ) 앱 관리 폴더로 복사
- 중복 처리
  - 자동 건너뛰기 (기본)
  - 확인 후 처리
- 지원 파일 형식 선택 (오디오/영상/이미지)

### 16.3 분석 / AI

- 자동 분석 사용 여부
- 분석 수준
  - 기본: 파일명/폴더명/메타데이터
  - 확장: Sound Analysis / Speech / Vision
  - 고급: 로컬 LLM
- 백그라운드 분석
- 외부 AI 사용 허용 (옵션)
- 분석 재실행 / 초기화

### 16.4 태그 / 카테고리

- AI 추천 태그 자동 적용 여부 (기본: 비활성, 추천만 표시)
- 사용자 태그 우선 적용
- 태그 색상 관리
- 기본 카테고리 편집
- 외톨이 기준
  - 사용자 카테고리 없음
  - 프로젝트 연결 없음
  - 확정 태그 없음

### 16.5 미리보기

- Space 미리보기 사용
- 선택 이동 시 자동 재생
- Hover 미리듣기
- 기본 볼륨
- 반복 재생
- 영상 미리보기 품질

### 16.6 Quick Peek (바로가기 돋보기)

- 전역 단축키 설정
- 기본 표시 항목
  - 최근 사용 / 즐겨찾기 / 최근 추가
- Enter 동작 (열기/복사/드래그 준비)
- Esc 동작

### 16.7 Final Cut Pro 연동

- 사운드 라이브러리 연결 상태 표시
- 소프트 링크 연결 / 해제
- 재연결/재생성
- 파컷 재시작 필요 안내
- 드래그 앤 드롭 동작 옵션

### 16.8 무음 제거

- 무음 감지 사용 여부
- 무음 판정 기준(앞/뒤 최소 길이)
- 클린 버전 생성 시 파일명 규칙 (`_trimmed`, `_clean`)
- 클린 파일 저장 위치
- 원본 보존 여부(기본: 보존)

### 16.9 저장소 / 링크 상태

- 등록된 경로 목록 표시
- 외장 드라이브 상태 표시
- 연결 끊긴 에셋 검사/재연결
- 누락 파일 숨기기 옵션

### 16.10 인터페이스

- 다크/라이트 모드
- 리스트 밀도(컴팩트/기본/넓게)
- 태그 칩 표시
- 파형 표시
- 우측 정보 패널 기본 상태

### 16.11 MVP 포함 범위

초기 버전에서는 아래 항목만 제공한다.

1. 에셋 추가 기본 동작(소프트/복사)
2. 중복 처리
3. 미리보기 관련 옵션
4. 분석/AI 수준 선택
5. Final Cut Pro 연동 상태 및 연결
6. 캐시/DB 관리
7. Quick Peek 전역 단축키

### 16.12 지원 전략


- Minimum
	- macOS 13 Ventura 이상
- Primary QA Target
	- macOS 15 Sequoia
	- macOS 26 Tahoe

**중요**

Tahoe는 최신 macOS 26이고, Apple 공식 지원 문서에도 호환 Mac 목록이 따로 제공된다.
Tahoe는 Intel Mac 지원이 크게 줄어든 버전이라, Apple Silicon 기준 최적화가 중요하다.
영상/음악 작업자는 OS 업그레이드에 보수적인 편이라, Sequoia 호환성도 반드시 안정적으로 가져가야 한다.

**기본 기능은 macOS 13 이상에서 동작하고, 고급 AI/분석 기능은 OS 및 하드웨어 지원 여부에 따라 선택적으로 활성화한다.**
