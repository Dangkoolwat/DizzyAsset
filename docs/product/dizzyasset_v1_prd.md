# DizzyAsset v1.0 PRD (Final)

## 1. 제품 정의

**DizzyAsset은 맥과 외장 SSD에 흩어진 편집 리소스를 복사 없이 인덱싱하고, 편집자 언어로 정리하여 Final Cut Pro에서 바로 사용할 수 있게 해주는 위치 독립 에셋 허브이다.**

---

## 2. 핵심 가치

1. 복사 없이 관리 (내장 / 외장 / 다운로드 어디든 OK)
2. 빠르게 찾는다 (검색 + 태그 + 카테고리)
3. 바로 쓴다 (Final Cut Pro 드래그)
4. 자동으로 정리된다 (분석 기반 초안 제공)

---

## 3. 핵심 사용자

- Final Cut Pro 영상 편집자
- 유튜브/쇼츠 제작자
- 밈 영상 편집자
- 효과음 중심 작업자

---

## 4. 시스템 구조

### 핵심 개념

파일은 이동하지 않고, 의미만 관리한다.

- 물리 파일: 내장 / 외장 / 다운로드
- 논리 구조: DB 기반 (태그, 카테고리, 프로젝트)

## 14. 경쟁 환경 및 UX 차용 전략

### 14.1 경쟁 구도

DizzyAsset은 특정 단일 경쟁 제품과 직접 경쟁하기보다, 여러 도구를 부분적으로 대체하는 구조를 가진다.

#### 직접 유사 제품
- Soundly
- SoundQ

특징:
- 효과음 라이브러리 중심
- 빠른 검색 및 미리듣기 제공

한계:
- 사용자 로컬 에셋 관리 기능 부족
- 외장 SSD 기반 워크플로우 미지원
- 커스터마이징된 분류 구조 제한

#### 간접 경쟁 제품

Finder
- 가장 많이 사용되는 파일 탐색 도구
- 검색, 정리, 미리보기 UX가 편집 작업에 최적화되어 있지 않음

Final Cut Pro 내장 라이브러리
- 기본 사운드 효과 제공
- 외부 에셋 관리 및 분류 기능 제한적

Eagle / Adobe Bridge / Kyno
- 강력한 미디어 관리 기능 제공
- 편집 워크플로우 연결성 부족

---

### 14.2 핵심 포지션

> DizzyAsset은 “에셋 관리 앱”이 아니라  
> 편집 워크플로우 도구이다.

---

### 14.3 차용해야 할 핵심 UX

#### 1. 초고속 탐색 UX (Soundly 계열)
- 검색 → 결과 표시 → 즉시 미리듣기
- 키보드 중심 탐색
- 낮은 지연시간

목표:
검색 후 0.3초 이내 미리듣기 시작

#### 2. Space 기반 미리보기 (Finder Quick Look)
- Space 키로 즉시 재생
- 방향키 이동 시 연속 미리듣기
- macOS 기본 UX와 동일한 경험

#### 3. 태그 기반 분류 UX (Eagle 일부)
- 태그 칩 UI
- 다중 태그 지원
- 색상 기반 태그
- 복잡한 관리 UI는 제외

#### 4. Final Cut Pro 드래그 UX
- 드래그 → 즉시 타임라인 삽입
- 중간 단계 없음

#### 5. Spotlight 스타일 Quick Search
- 전역 단축키 기반 실행
- 빠른 오버레이 UI
- 키보드 중심 탐색

---

### 14.4 지양해야 할 패턴

- 복잡한 폴더 기반 관리 구조
- 강제 라이브러리 복사 방식
- 과도한 자동 분류 (신뢰도 낮은 AI 태그)
- 관리 중심 UI

---

### 14.5 DizzyAsset의 차별화 요소

1. 외장 SSD 중심 설계
2. 편집자 언어 기반 분류
3. Final Cut Pro 중심 워크플로우 통합
4. 무음 구간 자동 감지

---

### 14.6 핵심 사용 흐름

검색 → 미리듣기 → 드래그 → 완료

이 흐름은 DizzyAsset UX 설계의 최우선 기준으로 유지한다.
