## Purpose
- Keep the canonical docs locations short and explicit.
- Reduce repeated discovery work for agents that need to know where plans, logs, workflows, and policy files live.
- Preserve the source docs as the truth; this file is only a cache.

## docs/ Tree Map
- `docs/agent-log/`: task logs and work reports
- `docs/agent-policy/`: execution rules and tool guides
- `docs/analysis/`: repo-local summaries and routing cache
- `docs/guidelines/`: technical guidelines
- `docs/knowledge/`: reusable lessons and notes
- `docs/product/`: PRDs and design docs
- `docs/superpowers/plans/`: execution plans for non-trivial tasks
- `docs/task/`: task specs and review reports
- `docs/templates/`: shared templates
- `docs/workflows/`: workflow checklists and handoff flow
- root docs: `AGENTS.md` and `history.md`

## Update Rules
- Update source policy docs first.
- Update this cache only after the source docs are settled.
- Keep the cache short; add deltas, do not rewrite history.
- Rebuild this file with `scripts/sync-docs-routing-cache.sh` after docs-tree changes.

## Cross-Repo Note
- Do not assume another repo uses the same `docs/` layout.
- Re-derive the folder map per repository before copying this pattern.
